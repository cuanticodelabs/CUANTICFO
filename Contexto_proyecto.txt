Sí. Para partir no necesito más datos; puedo avanzar con una planificación base usando supuestos razonables para CuantiCode Labs SpA, pensando en una app web responsive/mobile para uso interno de CFO.
El enfoque correcto no es construir “otro Nubox completo”, sino una herramienta CFO-first: captura, ordena, calcula, alerta y prepara información tributaria/financiera, pero dejando la presentación final ante SII con revisión humana.

1. Visión del proyecto
Nombre conceptual
CuantiCFO o CFO Control Center
Objetivo
Crear una aplicación web responsive que permita:
Centralizar ingresos, gastos, compras, ventas, honorarios y movimientos bancarios.
Generar libros contables mensuales/anuales.
Construir estado de resultados, flujo de caja y KPIs.
Preparar información para declaraciones mensuales en SII, especialmente IVA/F29.
Automatizar alertas y reportes mediante n8n.
Reducir trabajo manual y riesgo de errores antes de declarar impuestos.

2. Referentes investigados y aprendizaje clave
Los referentes más cercanos son Nubox, Defontana, QuickBooks y el propio ecosistema del SII.
Nubox posiciona su software contable en Chile como una solución para gestionar contabilidad, automatizar procesos, facilitar conciliación y generar reportes rápidamente; además permite manejar múltiples empresas desde una sola plataforma. (Nubox)
QuickBooks, por su parte, concentra su propuesta en seguimiento automático de ingresos/gastos, facturación, conciliación bancaria, presupuestos, gestión de impuestos y reportes financieros. (QuickBooks)
Para Chile, el SII indica que el Formulario 29 se usa para declarar y pagar impuestos mensuales como IVA, PPM y retenciones. (Servicio de Impuestos Internos) También tiene una guía oficial para declarar F29 por internet desde el menú de Impuestos Mensuales. (Servicio de Impuestos Internos)
Además, el SII dispone de herramientas asociadas a libros de compra y venta, incluyendo importador, estructuras, resúmenes y periodos sin movimiento. (Servicio de Impuestos Internos)

-- ACTUALIZADO v1.1 (2026-05-08): Migración de Supabase/PostgreSQL a MongoDB --
Base de datos: MongoDB (cuanticfo DB). Colecciones definidas en cuanticfo_schema.mongo.js.
n8n se conecta a MongoDB usando el nodo oficial MongoDB Driver (connection string MONGODB_URI).
Aislamiento multiempresa: filtrado por empresa_id a nivel de aplicación (no RLS).
IDs: strings UUID generados por la aplicación / n8n antes de insertar (_id: string).
Montos: tipo Decimal128 en MongoDB (bsonType: "decimal").
Periodo: siempre formato YYYY-MM, validado con regex en el schema.

n8n puede integrarse con Webhook y MongoDB para automatizar transferencia de datos y flujos ETL. Sus webhooks soportan métodos de autenticación como Basic Auth, Header Auth y JWT, lo que sirve para proteger endpoints llamados desde Next.js. (n8n Docs)
