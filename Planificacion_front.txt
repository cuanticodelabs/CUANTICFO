 Planeación del Frontend en Next.js
Next.js con App Router es adecuado para estructurar layouts, rutas, componentes server/client y navegación moderna. (Next.js) Además, las Server Actions permiten manejar mutaciones y formularios desde el servidor, aunque para este proyecto convendría usarlas con criterio y no reemplazar todos los webhooks de n8n. (Next.js)
5.1 Objetivo del frontend
Ser la interfaz móvil/desktop para:
Registrar movimientos.
Revisar documentos.
Ver KPIs.
Ejecutar cierres.
Descargar reportes.
Enviar procesos a n8n.

/app
  /login
  /dashboard

  -- MOVIMIENTOS
  /ingresos
    /[id]          -- Detalle / edición de ingreso
    /nuevo         -- Formulario crear ingreso
  /gastos
    /[id]
    /nuevo
  /honorarios      -- NUEVO: Boletas de honorarios recibidas
    /[id]
    /nuevo
  /transferencias  -- NUEVO: Movimientos bancarios para conciliación
    /[id]
    /nuevo
  /documentos      -- NUEVO: Repositorio central de archivos adjuntos

  -- CONTABILIDAD
  /libros
    /compras       -- Libro de compras mensual
    /ventas        -- Libro de ventas mensual
    /honorarios    -- Libro de honorarios mensual
  /cuentas-contables  -- NUEVO: Gestión del plan de cuentas / categorías

  -- IMPUESTOS
  /impuestos
    /f29           -- Preparador F29 (predeclaración asistida)
    /historial     -- RENOMBRADO (era /resumen-mensual): histórico de impuestos_mensuales

  -- REPORTES
  /reportes
    /estado-resultados
    /flujo-caja
    /kpis

  -- OPERACIONES
  /cierres
    /mensual       -- Checklist de cierre mensual guiado
  /alertas         -- Centro de alertas CFO

  -- CONFIGURACIÓN
  /configuracion
    /empresas      -- Datos de empresa(s)
    /usuarios      -- Gestión de usuarios y roles
    /categorias    -- Categorías contables (tipos y plan de cuentas)
    /proyectos     -- NUEVO: Gestión de proyectos / centros de costo

5.5 Diseño responsive/mobile
Para celular, priorizaría navegación inferior:
Inicio | Movimientos | Impuestos | Reportes | Más

Pantallas móviles críticas:
Dashboard resumido.
Crear gasto rápido (con CategorySelector y ProjectSelector).
Subir documento (con OCR sugerido en Fase 3).
Ver IVA estimado.
Revisar alertas.
Cierre mensual (checklist).
Crear honorario rápido (retención calculada automáticamente al ingresar monto bruto).

Nota de diseño:
  - Módulo "Libros contables" con tablas densas → mostrar versión resumida en móvil
    y tabla completa en desktop (usar responsive table con scroll horizontal en mobile).
  - "Transferencias bancarias" en móvil → lista agrupada por fecha, sin columnas extra.
